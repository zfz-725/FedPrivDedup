# 联邦学习隐私保护去重框架（FED）

## 1. 框架概述

联邦学习隐私保护去重框架是一个基于FED（Fast Entity Deduplication）技术的跨机构联邦协同去重系统，通过TLS 1.3加密传输实现安全的跨机构数据交互。

### 核心功能
- 跨机构联邦协同去重
- TLS 1.3加密传输
- GPU加速的特征生成和相似度计算
- 隐私保护机制（密文交换、桶级隔离、即时销毁）
- 中心化架构（主节点统筹管理）

## 2. 系统架构

### 中心化架构
- **联邦主节点**：负责全局统筹、数据汇总与计算、结果分发
- **机构本地节点**：负责本地语料处理、特征生成、本地去重

### 执行流程
1. **本地预处理**：各机构独立完成本地语料预处理、特征生成和本地去重
2. **主节点协同计算**：各机构向主节点提交密文候选集，主节点完成全局相似度匹配
3. **结果分发与本地去重**：主节点将去重结果分发到各机构，各机构完成最终去重

## 3. 核心技术

### 隐私保护机制
- **最小信息传输**：仅传输<无语义随机ID、密文签名向量、桶ID、匿名化机构标识>
- **桶级隔离**：按全局桶ID分类处理，同一桶的文档才进行跨机构相似度对比
- **即时销毁**：计算完成后立即销毁密文和明文签名，仅保留重复文档索引
- **轻量对称加密**：使用AES-256加密保护候选集传输
- **TLS 1.3加密传输**：确保数据传输安全
- **匿名机构标识**：使用匿名化机构标识保护机构隐私

### GPU加速
- 完全复用FED原版的GPU核心计算逻辑
- GPU并行加密/解密操作
- 主节点GPU集群并行计算
- 通信-计算重叠优化

### 通信协议
- 客户端和服务端之间使用简单的文本协议通信
- 支持TLS 1.3加密传输
- 批量传输同桶数据，减少网络I/O次数

## 4. 实现细节

### 服务端
- 监听8080端口
- 管理客户端连接
- 分发全局参数
- 接收加密候选集
- 执行GPU全局相似度计算
- 分发去重结果
- 客户端身份验证
- 数据大小限制

### 客户端
- 连接服务端获取参数
- 本地数据预处理
- 生成MinHash签名
- 执行本地去重
- 加密候选集
- 发送到服务端
- 接收去重结果

### 技术特点
- **模块化设计**：客户端、服务端、加密工具等模块分离
- **隐私保护**：实现了AES加密、无语义ID、即时销毁等隐私保护机制
- **GPU加速**：使用CUDA进行并行相似度计算
- **可扩展性**：支持多客户端接入，主节点可弹性扩容
- **动态参数**：服务端统一管理全局参数，客户端动态获取
- **桶级隔离**：按桶处理数据，处理完立即销毁，减少隐私泄露风险

## 5. 安装与使用

### 系统要求
- Ubuntu 20.04+ 或类似Linux系统
- CUDA 11.7+
- OpenSSL 3.0+
- C++17 编译器
- CMake 3.18+
- conda环境（fed_env）

### 安装步骤
1. 安装依赖库（OpenSSL、CUDA）
2. 克隆代码库
3. 激活conda环境：`conda activate fed_env`
4. 编译项目：`./build.sh`

### 使用方法
1. 启动服务器：
   ```bash
   conda activate fed_env
   export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/root/miniconda3/envs/fed_env/pkgs/cuda-toolkit/lib64
   ./build/fed_server
   ```

2. 运行客户端1：
   ```bash
   conda activate fed_env
   export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/root/miniconda3/envs/fed_env/pkgs/cuda-toolkit/lib64
   ./build/fed_client client1 org1 127.0.0.1 8080 test_data_all/test_data_large1
   ```

3. 运行客户端2（触发全局计算）：
   ```bash
   conda activate fed_env
   export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/root/miniconda3/envs/fed_env/pkgs/cuda-toolkit/lib64
   ./build/fed_client client2 org2 127.0.0.1 8080 test_data_all/test_data_large2
   ```

- 定期更新证书和密钥
- 配置合适的加密套件
- 使用硬件安全模块（HSM）存储密钥

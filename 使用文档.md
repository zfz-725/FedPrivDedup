# 联邦学习隐私保护去重框架使用文档

## 1. 框架概述

联邦学习隐私保护去重框架是一个跨机构的联邦学习隐私保护去重系统，基于FED（Fast Entity Deduplication）技术，通过TLS 1.3加密传输实现安全的跨机构数据交互。

### 核心功能
- 跨机构联邦协同去重
- TLS 1.3加密传输
- GPU加速的特征生成和相似度计算
- 隐私保护机制（密文交换、桶级隔离、即时销毁）
- 中心化架构（主节点统筹管理）

## 4. 配置说明

### 4.1 证书配置

框架使用自签名证书进行TLS 1.3加密传输。首次运行前需要生成证书：

```bash
# 生成CA证书、服务器证书和私钥
openssl genrsa -out ca.key 2048
openssl req -x509 -new -nodes -key ca.key -sha256 -days 365 -out ca.pem -subj "/CN=localhost"
openssl genrsa -out key.pem 2048
openssl req -new -key key.pem -out server.csr -subj "/CN=localhost"
openssl x509 -req -in server.csr -CA ca.pem -CAkey ca.key -CAcreateserial -out cert.pem -days 365 -sha256
```

### 4.2 服务器配置

服务器配置在代码中通过`ServerConfig`结构体设置，主要参数包括：

- `address`：服务器地址（默认：127.0.0.1）
- `port`：服务器端口（默认：8080）
- `max_clients`：最大客户端数（默认：10）
- `cert_file`：TLS证书文件路径（默认：cert.pem）
- `key_file`：TLS私钥文件路径（默认：key.pem）
- `use_tls`：是否使用TLS加密（默认：true）

### 4.3 客户端配置

客户端配置在代码中通过`ClientConfig`结构体设置，主要参数包括：

- `client_id`：客户端ID
- `org_id`：机构ID
- `server_address`：服务器地址（默认：127.0.0.1）
- `server_port`：服务器端口（默认：8080）
- `data_dir`：数据目录
- `output_dir`：输出目录
- `ca_cert_file`：CA证书文件路径（默认：ca.pem）
- `use_tls`：是否使用TLS加密（默认：true）

## 5. 使用方法

### 5.1 启动服务器

```bash
# 激活虚拟环境
export LD_LIBRARY_PATH=/root/miniconda3/envs/fed_env/lib

# 启动服务器
./fed_server
```

服务器启动后会监听在配置的地址和端口上，等待客户端连接。

### 5.2 运行客户端

```bash
# 激活虚拟环境
export LD_LIBRARY_PATH=/root/miniconda3/envs/fed_env/lib

# 运行客户端
./fed_client <client_id> <org_id> <server_address> <server_port> <data_dir>

# 示例
./fed_client client1 org1 127.0.0.1 8080 data
```

### 5.3 执行流程

1. **服务器初始化**：启动服务器，初始化TLS和其他组件
2. **客户端连接**：客户端连接服务器，获取全局参数和密钥
3. **本地处理**：客户端处理本地数据，生成签名和桶ID
4. **本地去重**：客户端执行本地去重
5. **密文传输**：客户端将加密后的候选集发送到服务器
6. **全局计算**：服务器执行跨机构相似度计算
7. **结果分发**：服务器将去重结果分发到各客户端
8. **本地处理**：客户端根据结果执行最终去重

## 6. TLS 1.3加密传输功能

### 6.1 功能说明

TLS 1.3加密传输功能为框架提供了安全的跨机构数据交互能力，确保数据传输过程中的隐私保护。

### 6.2 实现细节

- 使用OpenSSL库实现TLS 1.3协议
- 服务器和客户端之间的所有通信都通过TLS 1.3加密
- 支持证书验证（测试环境可跳过验证）
- 自动协商加密算法和密钥

### 6.3 安全特性

- 强加密算法：使用TLS 1.3支持的现代加密算法
- 前向 secrecy：每次连接生成新的会话密钥
- 双向认证：服务器和客户端相互验证身份
- 防窃听：所有数据传输都经过加密

## 8. 故障排除

### 8.1 TLS连接失败

- 检查证书文件是否存在且有效
- 检查服务器地址和端口是否正确
- 检查网络连接是否正常
- 查看服务器和客户端日志以获取详细错误信息

### 8.2 CUDA错误

- 检查GPU驱动是否正确安装
- 检查CUDA版本是否兼容
- 确保GPU内存足够

### 8.3 编译错误

- 检查依赖库是否正确安装
- 检查编译脚本是否正确
- 查看编译日志以获取详细错误信息

## 9. 性能优化

### 9.1 GPU加速

- 使用高性能GPU以获得最佳性能
- 调整批处理大小以充分利用GPU内存

### 9.2 网络优化

- 使用高速网络连接（如InfiniBand）

### 9.3 配置优化

## 10. 安全最佳实践

### 10.1 生产环境配置

- 使用正式CA签发的证书，而非自签名证书
- 启用完整的证书验证
- 定期更新证书和密钥
- 配置合适的加密套件

### 10.2 密钥管理

- 使用硬件安全模块（HSM）存储密钥
- 实施密钥轮换机制
- 严格控制密钥访问权限

### 10.3 网络安全

- 限制服务器访问权限
- 使用防火墙保护服务器
- 监控异常连接和数据传输

## 11. 常见问题

### 11.1 客户端无法连接服务器

- 检查服务器是否运行
- 检查网络连接是否正常
- 检查TLS证书是否正确

### 11.2 去重结果不准确

- 检查LSH参数是否合理
- 检查相似度阈值设置
- 确保各机构使用相同的全局参数

### 11.3 性能问题

- 检查GPU使用情况
- 优化批处理大小
- 考虑使用更强大的硬件
